# Narrator fine-tuning dataset

Цель: дообучить базовую модель на роли **narrator** — лаконичная, живая, **заземлённая**
русская проза вместо наигранной пурпурной отсебятины. Один адаптер на ВСЕ формы вызова
(общий system-промпт `PROMPTS["narrator"]`), которые различаются параметром **mode**
(вид реплики):

| mode      | где                                   | что генерит                          |
|-----------|----------------------------------------|--------------------------------------|
| dialogue  | `render_dialogue` (`_npc_reply`)       | реакция NPC + 1 реплика в кавычках    |
| greeting  | `render_dialogue` (`_npc_greeting`)    | короткое приветствие NPC              |
| outcome   | `render_scene` (`_narrate_outcome`)    | исход действия от 2-го лица           |
| combat    | `render_scene` (бой)                   | удар/промах строго по Outcome         |
| ambient   | `render_scene` (`discovery`)           | атмосфера места, без событий          |

**Вход параметризован** (единый билдер `agents.narrator_user(mode, …)`, его же импортирует
`build.py` → train == inference): `name` + вид NPC (`kind`=профессия/архетип, `race`,
`voice`, `traits`, `epithet`), `rel` (отношение), `scene` (сезон/время/погода/место),
`situation`, `player_line`, `intent`, `tone`, `outcome`, `facts`, `topic`. Печатаются
только заданные поля.

Один обучающий файл `narrator.jsonl`:
```json
{"messages":[
  {"role":"system","content":"<PROMPTS[narrator]>"},
  {"role":"user","content":"<narrator_user(mode, …) — РОВНО как на инференсе>"},
  {"role":"assistant","content":"<эталонная проза>"}
]}
```
Вывод — **проза**, не JSON. `build.py` валидирует СТИЛЬ, а не структуру.

## Что демонстрируют эталоны (assistant)
- **Лаконично:** 1–2 предложения для scene, 1–3 для dialogue. Без «воды».
- **Живо и конкретно:** образно, но просто; как у хорошего ведущего за столом.
- **Заземлено в сцене:** время суток / погода / сезон / место не противоречат `scene`
  (ночь → темно, без «первых лучей»; дождь → сыро; таверна → не «замок»).
- **Без пафоса:** запрещены клише вроде «сердце полно тепла», «словно сам фатум»,
  «добро пожаловать, странник!», «первые лучи солнца».
- **Имена дословно:** как переданы (Toblen Stonehill → так и оставлять); НЕ переводить,
  НЕ начинать реплику с имени самого говорящего (его подписывает интерфейс).
- **Никаких чисел/механики** в прозе scene (урон/DC/броски не печатать).
- **Реплика игрока** не выдумывается; общая история/факты — только из данных.

## Длина и ритм
Смесь коротких и длинных: рубленые punch-фразы соседствуют с более длинными описательными
предложениями (особенно ambient/прибытие/сложное действие). Длинные — тоже ПРОСТЫЕ и
приземлённые, без воды и пафоса. Чередовать, не дробить всё в обрывки.

## Конвенции (валидируются build.py)
- assistant непустой; длина по mode (dialogue/outcome ≤ 480, greeting/combat ≤ 360,
  ambient ≤ 560); ≤ 6 предложений.
- НЕ содержит цифр в outcome/combat/ambient (механику печатать нельзя).
- dialogue/greeting содержат ОДНУ реплику в кавычках «…».
- НЕ начинается с имени говорящего NPC; имя NPC, если упомянуто, — дословно.
- НЕ содержит фраз из блок-листа пафоса (см. build.py BANNED).
- Баланс mode и видов NPC (трактирщик/торговец/рыцарь/жрица/маг/дроу/кузнец-дворф/
  ребёнок/пленник/бандит), сцен (таверна/рынок/глушь/подземелье/храм/тракт/кузница/
  лес/поместье/комната), времени суток, погоды и тонов (дружелюбный/враждебный/
  боязливый/деловой/скрытный-лживый/умоляющий).
